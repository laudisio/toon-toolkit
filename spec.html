TOON Format Specification

TOON (Token Optimized Object Notation) is a compact representation of structured data designed to reduce token usage in Large Language Model (LLM) prompts.

Overview

JSON is widely used for structured data but can be verbose when used in prompts for AI models. TOON provides a more compact format that keeps structure while reducing unnecessary characters and tokens.

TOON is designed to be:

  • Human readable
  • Compact
  • LLM friendly
  • Easily convertible to and from JSON

Basic Object Example

JSON

{
  "user": {
    "id": 101,
    "name": "Alice",
    "email": "alice@example.com"
  }
}

TOON

user:
  id: 101
  name: Alice
  email: alice@example.com

Arrays

JSON

{
  "roles": ["admin", "editor"]
}

TOON

roles:
  [admin, editor]

Table Arrays (Optimized)

TOON introduces a compact table format for arrays of objects.

JSON

{
  "users":[
    {"id":1,"name":"Alice","role":"admin"},
    {"id":2,"name":"Bob","role":"editor"}
  ]
}

TOON

users[2]{id,name,role}:
1,Alice,admin
2,Bob,editor

Design Goals

  • Reduce token usage in AI prompts
  • Maintain structural readability
  • Allow deterministic conversion to JSON
  • Provide compact representations for repeated structures

Typical Use Cases

  • AI prompt optimization
  • Reducing OpenAI / LLM API costs
  • Compact structured context in prompts
  • LLM pipelines and agent systems

Compatibility

TOON is designed to be easily convertible to JSON. Tools like TOON Toolkit allow instant conversion between formats.
